Add in convert.h after QFactorResult:

```cpp
struct FactorResult {
    Frac content = Frac(1);             // rational constant (per FACTOR-02)
    int q_power = 0;                    // q^e factor (not cyclotomic)
    std::map&lt;int, int&gt; cyclotomic;     // n → exponent of Φ_n(q)
};
```

Implement `FactorResult factor(const Series&amp; f, int T)`:
1. qf = qfactor(f, T)
2. If qf.num_exponents.empty() &amp;&amp; qf.den_exponents.empty() and f has more than one nonzero coeff: throw "factor: cannot factor (not a q-product)"
3. result.q_power = qf.q_power
4. For each (n, e) in qf.num_exponents: require e integer (den==1); for each d in divisors(n): result.cyclotomic[d] += integer_part(e)
5. For each (n, e) in qf.den_exponents: same; result.cyclotomic[d] -= integer_part(e)
6. Remove entries with cyclotomic[d]==0
7. Return result

Use divisors(n) from qfuncs.h (already included). Handle Frac exponents: if an.den != 1, throw "factor: non-integer exponent in q-product" for robustness.
</action>

Build: make clean; make. Unit test: t8 = T_rn(8,8,50); auto fr = factor(t8, 50); fr.q_power == 6; fr.cyclotomic has positive entries for d in {5,8,9,10,11,16} (from T(8,8) expansion).
</verify>
  <done>FactorResult exists; factor(f,T) returns cyclotomic expansion from qfactor; throws on non-q-product or non-integer exponents.
</done>
