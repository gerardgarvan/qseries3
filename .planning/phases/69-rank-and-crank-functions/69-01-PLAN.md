## Goal

Implement `rankgf(m, T)` and `crankgf(m, T)` as built-in functions that return the generating function Σ N(m,n) q^n (rank) or Σ M(m,n) q^n (crank) as a truncated Series.

## Background

- **Rank** of a partition: (largest part) − (number of parts). Introduced by Dyson (1944).
- **Crank** of a partition: largest part if no 1s; μ − ω otherwise (μ = parts > ω, ω = number of 1s). Introduced by Andrews-Garvan (1988).
- Both satisfy symmetry: N(m,n) = N(−m,n), M(m,n) = M(−m,n).
- The crank resolves all three Ramanujan congruences (mod 5, 7, 11).

## Formulas

**Rank** (fixed m):
```
N(m,n) = δ_{m,0}δ_{n,0} + Σ_{j≥1} (-1)^{j-1} [p(n - j(3j-1)/2 - |m|j) - p(n - j(3j-1)/2 - |m|j - j)]
```

**Crank** (fixed m):
```
M(m,n) = Σ_{j≥1} (-1)^{j-1} [p(n - j(j-1)/2 - |m|j) - p(n - j(j-1)/2 - |m|j - j)]
```

where p(k) = partition number (p(k)=0 for k<0), computed via Euler's pentagonal recurrence.

Key difference: rank uses pentagonal k(3k−1)/2, crank uses triangular k(k−1)/2.

### Helper: compute_partition_numbers(T)

Both functions need partition numbers p(0)..p(T-1). Factor this into a shared helper to avoid code duplication:

```cpp
inline std::vector<Frac> compute_partition_numbers(int T) {
    std::vector<Frac> p(T, Frac(0));
    p[0] = Frac(1);
    for (int n = 1; n < T; ++n) {
        for (int k = 1; ; ++k) {
            int e1 = k * (3*k - 1) / 2;
            int e2 = k * (3*k + 1) / 2;
            if (e1 > n && e2 > n) break;
            Frac sign(k % 2 == 1 ? 1 : -1);
            if (e1 <= n) p[n] = p[n] + sign * p[n - e1];
            if (e2 <= n) p[n] = p[n] + sign * p[n - e2];
        }
    }
    return p;
}
```

### rankgf(m, T)

```cpp
inline Series rankgf(int m, int T) {
    if (T <= 0) return Series::zero(1);
    int am = (m < 0) ? -m : m;
    auto p = compute_partition_numbers(T);
    Series result;
    result.trunc = T;
    if (m == 0) result.setCoeff(0, Frac(1));
    for (int n = 1; n < T; ++n) {
        Frac val(0);
        for (int j = 1; ; ++j) {
            int kk = j * (3*j - 1) / 2 + am * j;
            if (kk > n) break;
            Frac sign = (j % 2 == 1) ? Frac(1) : Frac(-1);
            val = val + sign * p[n - kk];
            int kk2 = kk + j;
            if (kk2 <= n) val = val - sign * p[n - kk2];
        }
        if (!val.isZero()) result.setCoeff(n, val);
    }
    return result;
}
```

**Critical: the `if (m == 0)` correction.** The convolution sum gives N(0,0) = 0 (empty sum), but the correct value is 1 (empty partition has rank 0). For m ≠ 0, N(m,0) = 0 naturally.

### crankgf(m, T)

```cpp
inline Series crankgf(int m, int T) {
    if (T <= 0) return Series::zero(1);
    int am = (m < 0) ? -m : m;
    auto p = compute_partition_numbers(T);
    Series result;
    result.trunc = T;
    for (int n = 0; n < T; ++n) {
        Frac val(0);
        for (int j = 1; ; ++j) {
            int kk = j * (j - 1) / 2 + am * j;
            if (kk > n) break;
            Frac sign = (j % 2 == 1) ? Frac(1) : Frac(-1);
            val = val + sign * p[n - kk];
            int kk2 = kk + j;
            if (kk2 <= n) val = val - sign * p[n - kk2];
        }
        if (!val.isZero()) result.setCoeff(n, val);
    }
    return result;
}
```

**No δ correction needed for crank.** For m=0, j=1 gives kk = 0, so p(n-0) = p(n) enters the sum. At n=0: val = p(0) = 1, giving M(0,0) = 1 naturally. At n=1: the formula gives M(0,1) = −1, which is the standard Andrews-Garvan convention.

**Critical difference from rankgf:** The inner loop starts at n=0 (not n=1), and uses `j*(j-1)/2` instead of `j*(3*j-1)/2`.

#!/bin/bash
# acceptance-rank-crank.sh — Phase 69: Rank and Crank generating functions

    local desc="$1" input="$2" pattern="$3"
