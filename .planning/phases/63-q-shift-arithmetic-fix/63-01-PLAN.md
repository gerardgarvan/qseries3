<task type="auto">
  <name>Task 1: Add normalize_q_shift() and wire into arithmetic operators</name>
  <files>src/series.h</files>
  <action>
Add a `normalize_q_shift()` method to the Series struct that absorbs the integer part of q_shift into the coefficient index map, keeping q_shift in [0, 1). Place it after the `clean()` method (around line 97).

**normalize_q_shift() algorithm:**
1. Fast path: if `q_shift.isZero()`, return immediately (overwhelmingly common case).
2. Compute `F = floor(q_shift)` using `BigInt::divmod(q_shift.num, q_shift.den)`. Since divmod truncates toward zero, adjust for negative remainders: if remainder is non-zero and negative, subtract 1 from quotient to get true floor.
3. If F == 0, q_shift is already in [0,1), return.
4. Extract F as `int` from the BigInt quotient (safe — q_shift values are small in practice). Use the pattern: `int F = quot.d.empty() ? 0 : static_cast<int>(quot.d[0]); if (quot.neg) F = -F;`
5. Build a new `std::map<int, Frac>` with each key shifted by F: `new_c[e + F] = c[e]`.
6. Replace `c` with the new map.
7. Adjust `trunc += F`.
8. Set `q_shift = q_shift - Frac(BigInt(quot), BigInt(1))` to get the fractional remainder.
9. If `trunc <= 0` after adjustment, clear coefficients and set `trunc = 0`.
10. Call `clean()`.

**Wire into call sites — add `normalize_q_shift()` call:**

1. **operator\* (Series × Series)** — after `s.clean();` (around line 172), add `s.normalize_q_shift();` on the next line. This is needed because `q_shift = a.q_shift + b.q_shift` can produce values in [0, 2).

2. **inverse()** — add `result.normalize_q_shift();` before `return result;` in BOTH branches:
   - The `m != 0` branch (around line 210): add before `return result;`
   - The `m == 0` branch: add `g.normalize_q_shift();` before `return g;` (around line 227)
   This is needed because `q_shift = -q_shift` on a value in (0,1) produces a value in (-1, 0].

3. **subs_q(k)** — after `s.clean();` (around line 267), add `s.normalize_q_shift();`. This is needed because `q_shift *= k` can produce any value.

**Do NOT add normalize_q_shift() to:** `operator*` (Frac scalar), `operator-()` (unary), `truncTo()`, `withTrunc()` — these don't change q_shift. `pow()` is covered transitively through operator* calls. `operator+` just checks match.

**Do NOT change** the operator+ logic — per locked decision, it should still throw when q_shifts don't match (after normalization, compatible series will already have matching q_shifts).

**Part A: Rewrite str() for expanded fractional-exponent display.**

Replace the `str()` method (lines 293-336 of series.h) with a version that branches on `q_shift.isZero()`:

1. **When q_shift is zero** (the common case): use the EXACT existing display logic, unchanged. No prefix wrapping, no format changes. This is critical — per locked decision "integer-exponent series unchanged."

2. **When q_shift is non-zero** (fractional q_shift): display each term with its full exponent. Per locked decision, use "expanded individual terms" format:
   - For each coefficient at index `n`, compute `actual_exp = Frac(n) + q_shift`
   - If actual_exp is zero: display just the coefficient (e.g., `3`)
   - If actual_exp is integer (den == 1): use `q` + unicode superscript via `expToUnicode()`, e.g., `q²`. For exponent 1, just `q`.
   - If actual_exp is non-integer fraction: use parenthesized notation `q^(num/den)`, e.g., `q^(1/4)`, `q^(5/4)`.
   - Extract integer from actual_exp when den==1 using the same pattern as `fracExpStr`: `int v = actual_exp.num.d.empty() ? 0 : static_cast<int>(actual_exp.num.d[0]); if (actual_exp.num.neg) v = -v;`
   - Sign handling: first term shows leading `-` if negative; subsequent terms show ` - ` or ` + ` separator. Use `v.abs()` for the magnitude. This mirrors the existing sign logic.
   - Coefficient display: if `|coeff| == 1` and exponent is non-zero, omit the `1` (just show `q^(...)` not `1q^(...)`). If `|coeff| == 1` and exponent is zero, show `1`.
   - Per locked decision: O() term is always `O(q^trunc)` using integer `trunc`, same as existing: `" + O(q" + expToUnicode(trunc) + ")"`.

3. **Remove the old prefix wrapper** (`q^(frac) * (...)` format). The old `prefix` variable and wrapping logic are completely replaced by the expanded-term branch.
